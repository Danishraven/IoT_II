* eksperimenter og analyser skal vise de bedste tidsgrænser for hvor længe anordningen er ubeskæftiget, men aktiv.   
  * Efter nogle eksperimenter er vi kommet frem til at den bedste user experience er hvis der går 2 minutter fra at lyset slukker til at den går i deep sleep.  
  * User Story: As a group of users, then we want to press the button without the device falling into deep sleep. So that we get a better user experience without delay on the buttons.  
  * User case 2: As Button company A/S. Then we want the device to fall into deep sleep so then the device battery life cycle lasts longer before we have to go to the stores to replace the battery.  
* samt, hvordan skal opvågnen fungere, så anordningen opdage hvad der sker, og falder i søvn igen  
  * Button company A/S har beslutted sig at knappen kun skal lyse så længe du holder knappen nede  
  * User story: As an angry customer, I want to press the button multiple times in a row. So that I can show the store that I'm angry, and my voice is getting heard multiple times.  
  * User story 2: As a senior customer, I want to show my experience even if it takes some time to press the button, so that my voice counts as much as a younger person even if I press the button for longer.   
* en god analyse og overvejelse danner grundlag for hvor længe anordningen sover, inden den vågner rutinemæssigt.  
  * Button company A/S har beslutted at deviced kun skal starte op 1 gang klokken 2:00 hvor den skal sende alt data som den har fået i løbet af dagen klokken 2:00  
  * User story: As Button company A/S, We want to minimize our time frame of when we are open to get data, so that we can improve against cyber attacks.  
  * User story 2: As Button company A/S, we want to manage our load balance, so that we don't experience slow servers in the day because of all the packages we receive.  
  * User story 3: As Button company A/S, we want to make sure we only send data while the device is inactive, so that we don't lose button presses that would come in on the device while the device is sending data.
